## Prompt Injection and Malware Pattern Detection

GenRTL indexes documentation from public and private sources. To prevent malicious content from reaching AI assistants, GenRTL employs a **layered malicious content detection system**.

<Frame>
![How our detection system works](/images/security/classifier_pipeline.png)
</Frame>

- **Robust Detection** — Content is analyzed using a classifier tailored for GenRTL to identify prompt injection attempts and malware-related patterns
- **Targeted Validation** — Suspicious content is subjected to additional checks
- **Continuous Monitoring** — Flagged content is tracked and reviewed on an ongoing basis
- **Regular Updates** — Detection logic is updated to address evolving attack methods and new injection techniques

This ensures that documentation retrieved through GenRTL is safe to consume by both human developers and AI coding agents.
